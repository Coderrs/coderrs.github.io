Kubernetes and Monitoring

Kubernetes is a cloud-native, application-centric data protection platform that was designed from the ground-up to support the scale, performance and mobility requirements of Kubernetes container environments across any public or hybrid cloud environment.

Kubernetes offers backup and recovery of the entire application, including data, metadata and any other Kubernetes objects associated with the application so it is protected and able to be restored from any point in time.

Use Cases

  Backup & Recovery

Point-in-time backup and recovery to recover from data corruption Schedule automated incremental backups with pre-defined policies or on-demand. Selectively restore containers and applications to the same or new namespace.

Disaster Recovery

  • Build Disaster Recovery strategies leveraging backup data. Recover any Kubernetes application quickly in the event of a disaster.

Migration

  • Enable migration of your application and data between private and public clouds, or between public clouds to achieve better TCO or tighter control of your data. Migrate to a new Kubernetes cluster regardless of cloud infrastructure.

Application Mobility

  • Accelerate CI/CD by building multiple test/dev environments leveraging Trilio backups from a single target location. Create test/dev environments, validate code and push changes into production.
